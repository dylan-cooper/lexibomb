    Lexibomb Server v0.0.1-dev LexibombServer.Board.Grid

    Acts as the glue between the high-level board interface and the low-level square states.

    The grid module is responsible for establishing and manipulating the underlying data structure of a board’s coordinate system. The squares within a grid are referred to using 0-based indexes for both the rows and columns. These squares are stored within a MapSet using a {row, col} tuple as the key.

    An extra border of inactive squares is added to the a board’s size in order to simplify functions that would otherwise have to perform complicated boundary checks. This border means that a 15 x 15 sized board, will utilize a 17 x 17 grid underneath.
